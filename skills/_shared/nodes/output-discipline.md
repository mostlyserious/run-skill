# Shared Output Discipline

## Format

- Prefer concise, skimmable structure.
- Lead with the deliverable, then supporting notes.
- Keep language direct and concrete.

## File Handoff

- If a reply points the user to a file and they did not ask you to open it, end the message with the absolute path.
- Use `Path: /absolute/path/to/file` for a single file.
- Use `Paths:` followed by one absolute path per line for multiple files.
- Skip the path footer when no file is relevant to the reply.

## Quality

- Surface assumptions and unresolved risks.
- Avoid filler, hype, and generic conclusions.
- Match tone to context (executive, technical, conversational).
